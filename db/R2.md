# R2 对象存储（图标）

自定义图标文件存储在 Cloudflare R2 存储桶 `recohub-icons` 中，通过 `ICONS` 绑定访问。

## 存储规则

| 项目 | 说明 |
|---|---|
| Key 格式 | `{domain}.{ext}`（如 `www.example.com.png`），基于网站域名命名，同域名自动覆盖 |
| 文件大小限制 | 512 KB |
| 支持的格式 | PNG、JPEG、GIF、SVG、ICO、WebP |
| 缓存策略 | `Cache-Control: public, max-age=31536000, immutable` |

## 图标获取方式

| 方式 | 说明 |
|---|---|
| 自动获取 | 新增条目时服务端自动抓取 Favicon（`functions/lib/autoIcon.ts`），依次尝试解析 HTML `<link>` 标签、`/favicon.ico`、Google S2 API，成功后存入 R2 |
| 批量获取 | 图标管理中一键为所有 `icon_url IS NULL` 的条目获取图标，5 并发处理，流式 NDJSON 推送进度 |
| 手动选择 | 编辑弹窗中点击"获取图标"，从候选列表中选择，通过 `/api/icons/save` 下载并存入 R2 |
| 手动上传 | 编辑弹窗中上传本地图标文件，通过 `/api/icons` POST 接口存入 R2 |

## Key 生成规则

Key 由 `functions/lib/autoIcon.ts` 的 `getIconKey()` 函数统一生成，`icons/save.ts` 和 `icons/index.ts` 共享调用，基于 URL 的域名部分：

| URL | Key |
|---|---|
| `https://www.example.com/page` | `www.example.com.png` |
| `https://github.com/owner/repo` | `github.com.png` |

## 关联关系

- `items.icon_url` 和 `submissions.icon_url` 存储 R2 图标的访问路径（如 `/api/icons/www.example.com.png`）
- 删除单个图标时，所有引用该图标的 items 会自动将 `icon_url` 置为 NULL（恢复自动获取）
- 删除全部图标（`DELETE /api/icons`）会清空 R2 存储桶并将所有 items 的 `icon_url` 置为 NULL
- 数据导入时 `icon_url` 强制设为 NULL，因为图标文件不随数据导出
- 图标文件与数据库记录之间没有外键约束，通过 API 层维护一致性
